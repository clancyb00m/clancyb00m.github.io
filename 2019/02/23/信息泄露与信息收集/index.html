浅谈信息泄露与信息收集

发表于 2019-02-23 | 热度： °C

信息泄露

可根据右边导航栏进行快速浏览

概要

什么是信息泄露？

各种敏感信息的被公开或被黑客发掘出来

什么样的信息算泄露？

对攻击者有利的所有信息

分类

从web安全的角度来考虑

  1. web服务器信息：

    一般指的是类似apache、tomcat、iis、nginx等中间件的类型，版本，用来搜索是否符合某个漏洞

  2. web应用信息：

    web脚本类型（php,asp,jsp…）

    数据库类型（mysql,access,mssql,oracle,mongodb,redis,postgresql…）

    cms（dedecms,ThinkPHP,discuz!,wordpress…）

    内容相关信息（政府，金融，商城，博客…）

  3. 服务器操作系统信息：

    类型（Linux,Windows,Mac）

    版本（Centos 7,Ceontos 6,ubuntu,win 7,win 10…）

  4. 服务器其他服务信息：

    服务，端口，哪个端口对应哪些服务等

从被泄露对象的角度来考虑

  1. 个人信息泄露

    如身份证，电话号码，个人履历等，会被用于诈骗，钓鱼等

  2. 公司/组织信息泄露

    如商业计划，发展计划，科研项目等，可能会造成商业机密，国家机密泄露等危害

信息收集

简述

什么是信息收集？

针对目标，对其各种信息进行探测，包括可能造成信息泄露中的所有部分

信息收集的必要性

信息收集是为了给后续操作打基础，没有信息收集就好比盲人摸象

从web安全的方向来考虑，信息收集大致可分为：

域名收集，相关注册人信息收集，web应用信息收集，服务器信息收集（web服务器，服务器操作系统，服务器其他服务端口）

分类

域传送漏洞

DNS服务器分为主服务器，备份服务器，缓存服务器。备份服务器需要利用“域传送”从主服务器上copy数据，然后更新自身的数据库，以达到数据同步的目的，这样是为了增加冗余，万一主服务器挂了还有备份服务器顶着。而“域传送”漏洞则是由于dns配置不当，本来只有备份服务器能获得主服务器的数据，由于漏洞导致任意client都能通过“域传送”获得主服务器的数据（zone数据库信息）。这样，攻击者就能获得某个域的所有记录，甚至整个网络拓扑都暴露无遗，同时节省了信息收集的时间，还提升了准确度等等。参考自DNS域传送漏洞总结

总结

信息收集对于渗透测试来说可谓是重中之重！通过信息收集其实就有可能会发现一些存在的安全问题如发现某站使用了fckeditor编辑器，某站的cms是wordpress，那么可以尝试访问wp-login.php，再或者是某站是用的中间件版本刚好存在某种漏洞，网站中又存在上传点，即可直接getshell，正所谓“知己知彼，百战不殆”！
